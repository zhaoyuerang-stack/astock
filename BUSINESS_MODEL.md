# BUSINESS_MODEL — 商业模式设计(订阅 + Token 双轨)

> 定位:策略可行性审计 SaaS 的商业化设计。
> 收入 = 订阅服务费(月/年付,给基础 token 额度)+ Token 包(超出额度按量购买)。
> 本文档先于代码,定定价、定成本、定合规。

---

## 1. 核心商业逻辑

```
用户痛点:我不知道我的策略是不是真的有效(还是过拟合/成本吞噬/样本外失效)
平台价值:9-Gate 审计给出统计裁决,告诉你"可行/不可行/证据不足"
收费逻辑:审计消耗算力(回测+LLM),按次收费天然成立
```

**为什么订阅 + Token 双轨,而不是纯订阅或纯按次:**

| 模式 | 问题 |
|---|---|
| 纯订阅(月付无限用) | 重度用户刷审计,算力成本失控;轻度用户觉得贵不订阅 |
| 纯按次(¥X/次) | 用户心理门槛高,不敢第一次尝试;无续费粘性 |
| **订阅 + Token 双轨** | 订阅锁粘性(月度额度用不完不退,沉没成本)+ Token 包兜底重度需求 + 免费版拉新 |

## 2. 三档会员设计

### 免费版 FREE(拉新 + 转化漏斗)
- **价格**:¥0,永久免费
- **额度**:每月 3 token(约 3 次单因子审计)
- **功能限制**:
  - 仅单因子审计(动量/价值/小市值等单因子)
  - 基础 9-Gate 报告(无逐门深页)
  - 审计历史仅保留 7 天
  - 不含订阅消息推送
- **目的**:让用户体验核心价值,转化为付费

### 基础会员 PRO(主力营收档)
- **价格**:¥29/月,¥288/年(年付省 17%)
- **额度**:每月 30 token
- **功能**:
  - 单因子 + 双因子组合审计
  - 完整 9-Gate 报告 + 逐门深页
  - 审计历史永久保留
  - 审计完成订阅消息推送
- **目标用户**:有 1-3 个策略想验证的个人投资者

### 专业版 MAX(高 ARPU + 算力变现)
- **价格**:¥99/月,¥988/年(年付省 17%)
- **额度**:每月 120 token
- **功能**:
  - 多因子组合审计(≤5 因子)
  - 三段样本深审(IS/OOS/Stress)
  - 审计对比 + PDF 导出
  - 策略衰减告警 + 优先审计队列
- **目标用户**:量化爱好者、小型私募研究员

## 3. Token 消耗规则

**1 token = 1 次单因子审计的算力基准**

| 审计类型 | token 消耗 | 耗时 | 说明 |
|---|---|---|---|
| 单因子审计 | 1 | ~5min | 动量/价值/小市值/低流动性等 |
| 双因子组合 | 3 | ~10min | 动量+价值、小市值+低流动性 |
| 多因子组合(3-5) | 5 | ~15min | 5 因子加权 + 行业中性化 |

### Token 包(超出月度额度后购买)

| 包 | 价格 | 单价 | 说明 |
|---|---|---|---|
| 10 token 包 | ¥9 | ¥0.90/token | 轻度补额 |
| 30 token 包 | ¥25 | ¥0.83/token | 中度补额 |
| 100 token 包 | ¥79 | ¥0.79/token | 重度补额 |

- Token 包**不过期**,买完永久可用
- 月度额度**月底清零**(不累积,促使用户用完=养成使用习惯)
- Token 包优先于月度额度消耗?**否**,月度额度优先消耗(让用户先享受订阅权益)

## 4. 单次审计真实成本核算

### MVP 阶段(本地 Mac 回测,数据留本地)

| 成本项 | 金额 | 说明 |
|---|---|---|
| LLM 解析(DeepSeek) | ~¥0.02 | 自然语言 → spec,约 2K token in/out |
| 回测算力(本地 Mac) | ~¥0 | 已有硬件,边际成本 0 |
| 9-Gate 计算 | ~¥0 | CPU 计算,本地 |
| 推送 + 存储 | ~¥0.01 | 微信订阅消息 + JSON 存储 |
| **合计** | **~¥0.03/次** | |

### 收入侧(基础会员档)

- 基础会员 ¥29/月 = 30 token → **¥0.97/token**
- Token 包 ¥9/10 = ¥0.90/token
- **毛利率 ≈ 97%**(~¥0.93/次毛利)

### 升级到云回测后(方案 B/C,月成本 +¥50-80 服务器)

- 假设月活 100 用户,每人月均审计 20 次 = 2000 次/月
- 服务器成本 ¥80/月 ÷ 2000 = ¥0.04/次
- LLM 仍 ¥0.02/次
- **合计 ~¥0.06/次,毛利率仍 ≈ 94%**

**结论:这是个高毛利生意,核心瓶颈是获客成本和留存,不是算力成本。**

## 5. 续费与流失设计

### 粘性锚点:审计历史沉淀
- 用户每次审计的 spec + 9-Gate 报告永久留存(付费版)
- 形成个人"策略档案库",可对比、可回溯
- **流失成本高**:换平台 = 丢掉全部审计历史,重新验证一遍

### 召回钩子
- 流失用户每月 1 次免费审计(比免费版 3 次/月少,但够勾引回来看)
- 推送在册策略衰减告警(即使不付费也推,制造复访理由)
- 审计历史不删,降级后只读不可新建

### 续费激励机制
- 年付省 17%(¥288 vs ¥348)→ 锁定全年
- 连续订阅 3 个月送 10 token 包 → 培养使用习惯
- 邀请好友各得 5 token → 裂变

## 6. 微信支付合规要点(硬规则)

### 虚拟商品支付规则
- 小程序内虚拟商品/服务**必须走微信支付**,不能跳外部支付链接
- 微信支付手续费 0.6%(默认商户费率)
- 需要在微信支付商户平台开通"虚拟商品"类目

### 不得承诺(违反会下架)
- ❌ 不得承诺收益("用我们工具年化 20%")
- ❌ 不得承诺"包过审"("9-Gate 必过")
- ❌ 不得暗示投资建议("审计通过 = 该买了")
- ✅ 正确表述:"研究方法学工具,帮助您识别过拟合和样本外失效风险"

### 退款政策
- 7 天无理由退款(未使用 token 全额退)
- 已使用 token 按比例扣减后退款
- 月付随时取消不续费,当月权益用到月底
- 年付退款按剩余月份比例(已用月份按月付价差扣)

### 订阅协议必备条款
- 服务性质:研究方法学分析工具,非投资顾问服务
- 数据来源:公开市场数据,可能存在延迟或错误
- 结果性质:统计结论,不构成投资建议,不承诺未来表现
- 知识产权:审计方法学归平台,用户策略描述归用户

## 7. 收入预测(MVP 保守估计)

### 上线 3 个月(验证期)
- 月活 200 用户
- 付费转化率 8%(行业 SaaS 中位 5-10%)
- 付费用户 16 人,80% 基础 / 20% 专业
- 月订阅收入:16 × (0.8×29 + 0.2×99) = 16 × 43 = **¥688/月**
- Token 包收入(估 30% 付费用户超额):~¥150/月
- **合计 ~¥838/月**,覆盖服务器成本(¥80)后微利

### 上线 6 个月(增长期)
- 月活 800 用户
- 付费转化率 10%
- 付费用户 80 人
- 月订阅收入:80 × 43 = **¥3,440/月**
- Token 包收入:~¥600/月
- **合计 ~¥4,040/月**

### 上线 12 个月(成熟期目标)
- 月活 3000 用户
- 付费转化率 12%
- 付费用户 360 人
- 月订阅收入:360 × 43 = **¥15,480/月**
- Token 包收入:~¥2,500/月
- **合计 ~¥18,000/月**

**关键指标**:LTV/CAC > 3 才健康。订阅月费 ¥43,平均生命周期 6 个月 = LTV ¥258,意味着获客成本要控制在 ¥86 以内。

## 9. 与架构的集成点

### 新增后端模块
```
api/routers/payment.py           微信支付下单/回调/退款
api/routers/subscription.py      订阅状态查询/取消/续费
services/billing/token_ledger.py token 账本(消耗/余额/历史)
services/billing/plan_quota.py   套餐额度管理(月度重置/上限)
app_config/pricing.yaml          定价配置(档位/价格/token 规则)
```

### 数据存储
- `user_subscriptions`(订阅状态表):openid / plan / status / expire_at
- `token_ledger`(token 账本表):openid / delta / reason / balance / created_at
- `payment_orders`(支付订单表):order_id / openid / amount / product / status

### 审计流程集成
- 提交审计前:检查 token 余额,不足则引导购买
- 审计入队:预扣 token(防止并发超用)
- 审计失败:退还 token
- 审计完成:确认扣减 token

## 10. 风险与对策

| 风险 | 对策 |
|---|---|
| 用户觉得"审计 5-20 分钟太慢不值钱" | 教育市场:9-Gate 是机构级审计,快=敷衍;竞品没有 |
| 免费用户白嫖不转化 | 免费版 7 天历史清零 + 仅单因子,痛点够强才升级 |
| 微信支付审核卡住 | 提前准备虚拟商品资质 + 订阅协议 + 退款政策文档 |
| 算力成本随用户增长失控 | token 机制天然限流;重度用户用 token 包兜底,定价已含成本 |
| 付费用户流失(审计完就走) | 审计历史沉淀 + 衰减告警推送,制造持续复访理由 |
| 竞品抄袭 9-Gate | 9-Gate 是方法学不是代码,壁垒在审计库积累和数据 vintage |
