系统定位 & 不对称哲学

本平台是一套面向 A 股全市场的机构级因子自动搜寻、稳健性检验以及生产风控系统。核心策略哲学并非盲目最大化夏普比率，而是专注于构建“不对称收益结构”：

组件 不对称捕获机制 生产落地状态
illiquidity 因子 散户过度博弈下的流动性风险补偿 + 壳资源彩票溢价 LIVE (主力生产 v3.1)
PureTrend 择时 利用均线距离趋势跟踪天然截断左尾，砍掉暴跌 Regime LIVE (连续曝光 0-1.5x)
Salience Veto 风控 协方差一票否决剔除末端 30% 异常爆雷股/过度拥挤股 LIVE (防雷风控层)
511010 国债轮动 熊市防守期将空仓资金轮入债券，获取纯 carry 增益 LIVE (跨资产轮动)

9-Gate Systematic Risk Audit

任何被提议的新因子或新策略，必须无条件通过以下 **9-Gate** 顶级量化审计门禁，方可准入模拟盘或实盘：

00 Data Audit 数据审计 PASS
空值率/极值/数据穿越测试 NaN < 20% | Outliers < 0.1%
01 Economic Thesis 经济假设 PASS
拒绝纯随机挖掘，检验学术逻辑 Mechanic > 50 chars + Citation
02 Single Factor 因子检验 PASS
Rank IC/NW-ICIR/五分位单调 Rank IC > 0.03 | MonotonicITY = 1.0
03 Neutralization 中性化 PASS
Barra 行业与 Size 风格剥离 ICIR Retention > 30%
04 Multi-Testing DSR 惩罚 FAIL
试错惩罚，防过拟合自欺 DSR p-value < 0.05 | PSR > 95%
05 Backtest 组合回测 FAIL
裸因子多头真实费后净值 Return > 15% | MaxDD < 20%
06 Cost & Capacity 冲击建模 WARN
成本滑点衰减与 ADV 曲线 Net Sharpe > 0.5 @ AUM scale
07 Regime Stress OOS 压力 WARN
OOS滚动胜率与极端牛熊表现 Delay 1d Sharpe > 0.5
08 Live Monitoring 监控模型 PASS
期望边界监控与硬止损阈值 Tracking Error < 5%

AutoResearch 自动寻优与反思进化

系统每周日凌晨 2:30 定时通过 launchd 自动调用 scheduled_factor_search.py 进行因子的自进化挖掘，其闭环引擎工作如下：

  • LLM 播种与反思 (Failure Ledger)：DeepSeek 苦力层结合历史失败台账（包含历次死因如 L0 截面无信息、L1 换手率太高被成本吃光），避开已被证伪的逻辑空间，提出全新概念。
  • 多岛屿进化 (Multi-Island Evolve)：多岛并行演化，基于真实 A 股面板运行 Rank IC/ICIR/新颖度(novelty)综合适应度打分，进行精选和杂交变异。
  • 自动化 9-Gate 提审：通过 L3 OOS 验证的因子，一键转化为真实 9-Gate 评估，将结果 Markdown 报告推送到前端评审台。

stock_cli.py --code

研究工具箱 (Research Toolkit)

存放于 research_toolkit/ 的底层核心量化测谎与规则处理引擎，是系统策略质量的保护伞：

Alpha Audit

测谎机制 (alpha_audit.py)

通过 Newey-West 校正重叠期收益相关性，并基于 RidgeCV 联合非线性拟合和随机置换测试（Permutation test），隔离 NaN 模式对预测精度的干扰，给因子颁发 REAL, TRUE_BUT_SMALL, 或 NOISE 的真理 verdict。

Veto & Marginal

策略风控与边际贡献 (marginal.py)

支持在一级宿主选股前切入 apply_veto_filter，并提供 regime 敏感型的边际 ΔSharpe 分析，判断多因子合成后是否具备净特质阿尔法，避免因子“换皮”导致的伪多样性风险。

数据资产湖架构

通过 Tushare 数据接口与 TDX 通达信本地抓取整合，我们在 data_lake/ 下维持了一个包含 7100万行 数据的全方位 A 股底层数据库：

  • 数据对齐：所有数据根据 by_date (不 shift 纯历史) 与 anndate (根据财报/业绩快报的真实公告日向后 ffill，防未来穿越) 严格物理归拢。
  • 安全机制：写路径拦截器 lake/invariants.py 自动监测并拦截单日跌幅超过 50% 占比大于 5% 的假崩盘数据写入。
