# OPERATING_COST — 运营成本核算手册

> 定位:策略审计 SaaS 的完整运营成本模型,用于定价验证、盈亏平衡测算、融资叙事。
> 口径:月度成本,含 MVP / 成长期 / 成熟期三阶段预估。
> 本文档先于代码,定成本结构、定计算公式、定监控指标。

---

## 1. 成本结构总览(8 大类)

```
┌─────────────────────────────────────────────────────────┐
│  1. 服务器(CVM/轻量)    5. 域名 + SSL 证书              │
│  2. 对象存储(COS)        6. 微信支付手续费               │
│  3. LLM Token 消耗       7. 监控 + 日志 + 告警           │
│  4. 网络流量(带宽/CND)   8. 合规 + 备案 + 资质           │
└─────────────────────────────────────────────────────────┘
```

**成本特性分类:**

| 类型 | 项目 | 随用户量变化 |
|---|---|---|
| 固定 | 服务器、域名、证书、备案、监控 | 否(阶梯式跳变) |
| 变动 | LLM token、COS 存储、网络流量、支付手续费 | 是(线性/非线性) |
| 半固定 | 微信小程序认证费(年付) | 否(年度) |

## 2. 逐项计算公式

### 2.1 服务器(CVM/轻量云)

**公式:**
```
月服务器成本 = 实例规格月费 × 实例数 + 系统盘月费 + 数据盘月费
```

**腾讯云轻量服务器参考价(2026):**

| 规格 | CPU/内存 | 系统盘 | 月费 | 适合阶段 |
|---|---|---|---|---|
| 2C2G | 2核2G | 50GB SSD | ¥50 | MVP(<100 用户) |
| 2C4G | 2核4G | 60GB SSD | ¥80 | 成长期(100-1000) |
| 4C8G | 4核8G | 80GB SSD | ¥180 | 成熟期(1000+) |
| 8C16G | 8核16G | 100GB SSD | ¥400 | 重负载(回测上云) |

**MVP 选 2C2G ¥50/月**(只跑 FastAPI + Nginx + SQLite,回测在本地 Mac)
**成长期升 2C4G ¥80/月**(加 PostgreSQL + Redis + 队列 worker)

**数据盘**(存 audit_history + token_ledger + 用户数据):
- MVP:20GB ¥10/月
- 成熟期:100GB ¥40/月

### 2.2 对象存储(COS)

**公式:**
```
月COS成本 = 存储费(GB×单价) + 请求费(次数×单价) + 流量费(下行流量×单价)
```

**腾讯云 COS 参考价:**
- 存储:¥0.099/GB/月(标准存储)
- 请求:¥0.01/万次(读) / ¥0.01/万次(写)
- 下行流量:¥0.5/GB(外网) / ¥0.15/GB(内网到 CVM)

**计算(MVP):**
```
存储:data_lake 副本不传(34GB 留本地),只存 audit_history JSON
  假设每月 200 次审计 × 50KB/份 = 10MB/月 → 累积 1 年 = 120MB ≈ 可忽略
请求:200 次/月 × 2(读写) = 400 次 → ¥0.0004 ≈ 可忽略
流量:200 次 × 50KB = 10MB → ¥0.005 ≈ 可忽略

MVP COS 成本 ≈ ¥1/月(免费额度内)
```

**计算(成熟期,3000 用户):**
```
存储:3000 用户 × 月均 20 次审计 × 50KB × 12 月 = 36GB → ¥3.6/月
请求:3000 × 20 × 2 = 120,000 次 → ¥0.12/月
流量:120,000 × 50KB = 6GB → ¥3/月(内网)

成熟期 COS 成本 ≈ ¥7/月
```

### 2.3 LLM Token 消耗(最大变动成本)

**这是运营成本的核心,必须精确建模。**

**公式:**
```
月LLM成本 = 月审计次数 × 单次审计LLM消耗
单次审计LLM消耗 = 解析阶段token成本 + (可选)解释阶段token成本
```

**DeepSeek V4 定价(2026):**
- 输入:¥0.001/千 token(缓存命中 ¥0.0001)
- 输出:¥0.002/千 token

**单次审计 LLM 消耗拆解:**

| 阶段 | 输入 token | 输出 token | 成本 |
|---|---|---|---|
| 解析(用户描述→spec) | ~800 | ~400 | ¥0.0008 + ¥0.0008 = ¥0.0016 |
| 解释(9-Gate→人话) | ~1500 | ~300 | ¥0.0015 + ¥0.0006 = ¥0.0021 |
| **合计** | | | **~¥0.004/次** |

**按用户量预估月 LLM 成本:**

| 阶段 | 月活 | 付费率 | 付费用户 | 人均月审计 | 月审计次数 | 月 LLM 成本 |
|---|---|---|---|---|---|---|
| MVP | 200 | 8% | 16 | 10 | 160 | ¥0.64 |
| 成长 | 800 | 10% | 80 | 15 | 1200 | ¥4.80 |
| 成熟 | 3000 | 12% | 360 | 20 | 7200 | ¥28.80 |

**LLM 成本极低**(因为 DeepSeek 便宜),不是瓶颈。

**⚠️ 风险:如果切到 GPT-4o**
- GPT-4o 输入 $2.5/百万 token,输出 $10/百万 token
- 单次审计成本:¥0.03 + ¥0.02 = ¥0.05/次(贵 12 倍)
- 7200 次/月 = ¥360/月(成熟期)
- **结论:坚持用 DeepSeek,不切 GPT-4o**

### 2.4 网络流量(带宽/CDN)

**公式:**
```
月网络成本 = 固定带宽费 + 超额流量费 + CDN流量费(如有静态资源)
```

**腾讯云带宽参考价:**
- 按量计费:¥0.8/GB(公网出)
- 按带宽计费:1Mbps ¥25/月(适合稳定小流量)

**计算(MVP):**
```
200 用户 × 日均 5 次请求 × 10KB/响应 × 30 天 = 3GB/月
按量:3GB × ¥0.8 = ¥2.4/月
建议:用按量计费,不上固定带宽
```

**计算(成熟期):**
```
3000 用户 × 日均 10 次 × 20KB × 30 天 = 180GB/月
按量:180GB × ¥0.8 = ¥144/月
建议:升级 5Mbps 固定带宽 ¥125/月 + 超额按量
```

## 4. 盈亏平衡分析

### 单用户经济模型(Unit Economics)

**成熟期单个付费用户:**

```
收入:¥45/月(Pro ¥29 + Token 包均摊 ¥16)
成本:
  - LLM:¥28.80/7200次 × 20次 = ¥0.08
  - 服务器分摊:¥757.60/360 = ¥2.10
  - 网络分摊:¥125/360 = ¥0.35
  - 支付手续费:¥45 × 0.6% = ¥0.27
  - 监控分摊:¥250/360 = ¥0.69
  - 其他分摊:¥3.16/360 = ¥0.01
  - 合计成本:¥3.50/用户/月

毛利:¥45 - ¥3.50 = ¥41.50/用户/月
毛利率:92%
```

**盈亏平衡点:**
```
固定成本(成熟期)= ¥757.60 - (¥3.50 × 360) = ¥757.60 - ¥1260 = 负值
结论:成熟期 360 付费用户已远超盈亏平衡(因 LLM 成本极低)
```

**真实盈亏平衡(含获客成本 CAC):**
```
假设 CAC = ¥50(微信生态内投放+裂变)
LTV = ¥41.50 × 6 月(平均生命周期)= ¥249
LTV/CAC = 4.98 > 3 ✓ 健康

盈亏平衡付费用户数 = 固定成本 / 毛利
  MVP:¥97 / ¥41.50 = 2.3 用户(几乎立刻盈亏平衡)
  成熟期:¥757.60 / ¥41.50 = 18 用户
```

**结论:这是极轻资产生意,盈亏平衡点极低,核心瓶颈是获客不是成本。**

## 6. 成本优化策略

### 6.1 LLM 优化(最大优化空间)
- **缓存 spec 解析**:相似描述复用缓存,减少 LLM 调用
- **批处理解释**:非实时审计的 9-Gate 解释用批量 API(便宜 50%)
- **降级模型**:简单策略用 DeepSeek-flash,复杂组合才用 V4-pro

### 6.2 服务器优化
- **竞价实例**:非关键任务(如历史审计重算)用竞价实例,省 70%
- **自动伸缩**:白天扩容,夜间缩容(审计任务多在白天)
- **预留实例**:长期稳定负载买包年,省 30%

### 6.3 网络优化
- **内网通信**:COS ↔ CVM 走内网,省 70% 流量费
- **Gzip 压缩**:API 响应启用 gzip,省 60% 带宽
- **CDN 静态资源**:图表库/字体走 CDN,不占 CVM 带宽

### 6.4 回测算力优化
- **MVP 不上云**:回测在本地 Mac,¥0 算力
- **成熟期按需上云**:用户审计触发时才启容器,用完释放
- **预计算**:常见因子组合预先算好缓存,命中即返回

## 8. 成本计算器(快速复用)

```python
# 运营成本计算器 - 改参数即可复用
def calc_monthly_cost(mau, paid_rate, avg_audit_per_user, avg_spend):
    """计算月运营成本"""
    paid_users = mau * paid_rate
    monthly_audits = paid_users * avg_audit_per_user
    
    # LLM 成本(DeepSeek)
    llm_cost = monthly_audits * 0.004  # ¥0.004/次
    
    # 服务器(阶梯)
    if mau < 500:
        server = 60  # 2C2G + 盘
    elif mau < 2000:
        server = 100  # 2C4G + 盘
    else:
        server = 220  # 4C8G + 盘
    
    # 网络
    traffic_gb = mau * 10 * 0.02 * 30  # 日均10次×20KB
    network = min(traffic_gb * 0.8, 125)  # 按量 vs 固定取小
    
    # 支付手续费
    payment_fee = paid_users * avg_spend * 0.006
    
    # 监控
    monitoring = 0 if mau < 500 else 250
    
    # 固定(域名+合规)
    fixed = 30  # 域名+认证摊销
    
    total = llm_cost + server + network + payment_fee + monitoring + fixed
    
    # 收入
    revenue = paid_users * avg_spend
    
    return {
        'mau': mau,
        'paid_users': int(paid_users),
        'monthly_audits': int(monthly_audits),
        'llm_cost': round(llm_cost, 2),
        'server': server,
        'network': round(network, 2),
        'payment_fee': round(payment_fee, 2),
        'monitoring': monitoring,
        'fixed': fixed,
        'total_cost': round(total, 2),
        'revenue': round(revenue, 2),
        'profit': round(revenue - total, 2),
        'margin': f"{(revenue - total) / revenue * 100:.1f}%"
    }

# 三阶段验证
for scenario in [
    ('MVP', 200, 0.08, 10, 35),
    ('成长', 800, 0.10, 15, 40),
    ('成熟', 3000, 0.12, 20, 45),
]:
    name, mau, rate, audits, spend = scenario
    r = calc_monthly_cost(mau, rate, audits, spend)
    print(f"\n=== {name} ===")
    for k, v in r.items():
        print(f"  {k}: {v}")
```
