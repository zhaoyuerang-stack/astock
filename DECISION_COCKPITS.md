# DECISION_COCKPITS.md — 决策驾驶舱架构

> 本文是 Web 端的**决策层纲领**:定义这套系统服务哪些决策、每个决策对应哪个驾驶舱、读哪个数据源、当前缺口在哪。
> 它**约束并重排** `WEB_DESIGN.md` 的页面实现:`WEB_DESIGN.md` 定义"三栏布局/组件/视觉规格",本文定义"页面为什么存在、回答什么决策"。冲突时,决策归属以本文为准,布局/组件以 `WEB_DESIGN.md` 为准。
> 上游依据:`CLAUDE.md §1`(母策略生命周期 = 系统操作循环)、`WEB_DESIGN.md §1.2`(研究优先,不以交易为中心)。
> 落地排期与现状以 `Implement.md` / `STATUS.md` 为准。

---

## 0. 一句话

**前端不是"展示数据的看板",是"沿 alpha 生命周期驱动决策的驾驶舱"。每个视图 = 一个决策 + 支撑它的证据。**
为填满页面而产数据,违反本文;为某个决策而产/接数据,才是本文。

---

## 1. 第一性原则:数据为决策而用

判断一个页面"是否健康",不看它满不满,看它服务的**那个决策能否被做出来**:

| 页面状态 | 含义 | 处理 |
| --- | --- | --- |
| 空,因为答案是"否" | 完整的决策支持(如:今天不能交易,因 0 在册 + fail-closed) | **正确**,不得填假数据 |
| 空,因为"该算的证据从没算" | 真缺口 | 立项补算/接线 |
| 满,但全是排行榜/收益曲线 | 诱导过拟合的 vanity metrics | **反模式**,见 §6 |

推论:撤掉 mock 绿灯后页面"变空",绝大多数是第一类(诚实),不是退步。

---

## 2. 用户是谁:一人轮值的三道防线

用户 = 一个 **AI 增强的单人 A 股量化 shop 负责人**,轮值机构里的四个角色;AI agent 辅助但**不做有效性裁决**(`R-LLM-001`)。顶级机构(AQR / Two Sigma / Citadel / Millennium / WorldQuant)以**三道防线 + 受治理流水线**运营,非"指标看板":

| 防线 | 角色 | 本仓已有能力 |
| --- | --- | --- |
| 第一线 | Researcher(找 alpha) + PM(部署资金、承担风险) | `factory/` `strategies/` `portfolio/` |
| 第二线 | 独立模型验证 / 风控(Fed SR 11-7) | `model_risk/` `core/analysis/nine_gates.py` |
| 第三线 | 审计治理(不可篡改台账 + 签名) | `research_ledger/`(hash-chain) `strategy_registry` |

机构的"系统状态"= 一句话:**每个候选处在生命周期哪一段,卡它前进的那一个决策是什么**。

## 3. 阶段定位(决定驾驶舱重心的最重要判断)

当前真实状态(主 checkout 实测,2026-06):
- 价格数据新鲜(到 2026-06-25),是可信地基。
- 策略 44 个:21 参考 / 15 候选 / 7 退役 / 1 已证伪、**在册 = 0**。
- 候选 40 个、研究台账 1494 条、逐版 DSR/PBO/holdout 已算。
- 部署 fail-closed(声明 `illiquidity/v3.1` 已降为"参考")、decay = red(且报告陈旧、列的是已退役腿)、paper 账户 blocked。

**行业判断:这是一家"首次部署前"的研究 shop。0 在册、fail-closed 是健康、正确的状态。**
因此驾驶舱重心**必须**是"alpha 工厂 + 验证闸门",而**不是**带空 P&L 的交易台。在没有验证过的 alpha 之前:
- 注意力重心 = 第一线研究 + 第二线验证;
- 交易台诚实显示"资金在板凳上",不造收益曲线。

## 4. 决策驾驶舱地图(沿生命周期)

`CLAUDE.md §1` 生命周期:`发现 → 证伪 → 入册 → 监控 → 退役 → 替换`。四个驾驶舱 + 一个地基 + 配置,每个只回答一个决策。

### 驾驶舱 ① Alpha 工厂(研究员)— 当前价值重心
- **决策**:下一轮研究算力 / 晋级注意力投向哪?哪个候选值得推进到下一道门?
- **生命周期段**:发现 → 证伪。
- **首屏(答案先行)**:不是按年化排行的排行榜,而是
  1. **晋级就绪榜**:候选按"距入册还差几道门"排序,每行给**唯一卡点门 + 边际动作**(例:`small-cap-size/v2.0:DSR p=0.086,差显著性;在小盘簇继续微调边际≈0,建议换信号源`);
  2. **信息地图 / 拥挤度**:候选映射到信息簇(价格/成交额/基本面/资金流/事件/微结构),显示冗余率与"最缺信号簇";
  3. **研究预算**:每个主题已试候选数 vs 独立信息簇数 vs 当前最优 DSR vs 继续搜索的边际,给"停/转"建议。
- **数据源**:`/experiments/funnel`、`/experiments/autoresearch/candidates`、registry 逐版 `nine_gate`(DSR/PBO/PSR/n_trials)、`research_ledger`(1494 条)。
- **归并现有页**:`factor-research` + `experiments` + `backtest-lab` + candidates → 一条漏斗。
- **现状/缺口**:原料全有,**缺"合成为决策"的透镜**(信息地图、距离-入册、边际动作)。这是第一优先建设面。
- **反模式**:Sharpe 排行榜首屏、只展示漂亮回测曲线。

### 驾驶舱 ② 验证闸门(第二线 · 模型验证/风控)
- **决策**:这个候选能否**独立验证**通过 → 入册?在册策略是否仍合规?
- **生命周期段**:入册 gate。
- **首屏**:候选/版本的 **9-Gate R2P 裁决**(逐门 PASS/FAIL + 证据),DSR 多重检验惩罚、PBO、holdout 合规、model card(SR 11-7)、独立验证结论(独立于研究,不自评)。
- **数据源**:`/governance`(model_cards 47 / validation 47 / ledger)、registry `nine_gate`、`/system/truth`(部署身份)、`holdout_compliance`。
- **归并现有页**:`strategy-registry`(生命周期/9 门状态) + `system-governance`(部署真相/CI 守卫/依赖拓扑/审计)。
- **现状/缺口**:引擎与证据已落台账,**缺"作为一次验证决策"的呈现**(逐门裁决面、"为什么这个候选还没入册"的单一根因)。`system-governance` 已接 `/system/truth`(部署真相),CI 守卫表/治理九宫格仍部分硬编码,待接真实守卫运行结果。
- **铁律**:研究不得自评;验证把守入册(`R-EVIDENCE-001` / `R-OBJECTIVE-001`)。

### 驾驶舱 ③ PM 交易台(PM)— 当前诚实=板凳
- **决策**:今天交易吗?用什么?多大风险?需要谁签字?
- **生命周期段**:部署 → 监控 → 退役。
- **首屏**:`production_allowed` 三态(declared / verified / production_allowed)+ 阻断证据链(已由 `DeploymentTruth` 实现);若有在册可部署标的:目标组合、预期风险/容量、需人工签字项;decay/退役告警。
- **数据源**:`/system/truth`、`/trade-readiness`、`/portfolio`、`/signals`、`/paper/*`、`reports/decay_status.json`。
- **归并现有页**:`dashboard`(今日操作台) + `portfolio-risk` + `signal-audit`。
- **现状**:`DeploymentTruth` 已诚实回答"今天=否,因 0 在册 + fail-closed"。decay 报告陈旧需重算才能正确退役监控。**不得**为填台面造 P&L。

### 地基 ④ 数据与成本可信层(横切,一切之下)
- **决策**:我能信这些数吗?口径对吗?
- **首屏**:最新交易日对齐、PIT 通过率、OHLC 质量、成本模型口径与变更。
- **数据源**:`data_lake/_manifest.json`、`quality_report.json`、`reports/data/data_issue_triage.json`、`cost_model.md`。
- **归并现有页**:`data-health`。优先级链最底:`数据可信 > 假设清晰 > …`(`WEB_DESIGN.md §1.2`)。

### 配置 ⑤ `settings`
- 成本/风控阈值/LLM key。保持现状。

## 6. 重组必须落实的顶级实践原则(每条可验收)

1. **答案先行**:每页开屏给"决策 + 裁决",证据点开看。验收:首屏不靠用户拼指标即可读出结论(`DeploymentTruth` 已示范)。
2. **单一真相源 + 全程可追溯**:每个数字 → 文件 / hash / 数据版本。验收:任一绿/红灯可追溯到 registry / deployment / spec_hash / data vintage / readiness(把 `/system/truth` 模式推广全站)。
3. **硬门禁 + 独立验证**:研究不得自评,验证独立把守入册。验收:页面 PASS 必须等同台账裁决,杜绝"页面 PASS / 台账 FAIL"脱节。
4. **alpha = 一篮子会衰减的信号**:按 family 生命周期组织,不按版本号。验收:每个 family 显示血缘 + 增量 alpha + 失效信号。
5. **对抗式研究(guilty until proven)**:多重检验惩罚(DSR/PBO)摆首屏,容量/拥挤前置到 L1/L2。验收:候选排序含 DSR 与拥挤度,而非裸 Sharpe。
6. **永远指出"约束在哪"**:把注意力路由到卡住下一步的**唯一**那件事。验收:每个驾驶舱首屏有一行"当前唯一瓶颈 + 边际动作"。

**反模式清单(禁止)**:空 P&L 曲线充门面、按年化/Sharpe 的诱导排行榜、硬编码绿灯、把 reference/候选当 live、AI 直接裁决有效性或越权发信号(`R-LLM-001`)。
