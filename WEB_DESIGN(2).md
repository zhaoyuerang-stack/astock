# WEB_DESIGN.md

# 个人策略信号与交易计划助手｜Web 端设计规格

版本：v3.0 Signal & Trade Plan First  
适用项目：我的 Alpha 工作台 / Personal Alpha Factory OS  
适用范围：Web 端产品设计、前端实现、右侧 AI 助手、信号/候选/交易计划/提醒交互、P0/P1 页面边界  
核心原则：表层解决“哪些股票、什么价格、何时提醒”，底层保留 Alpha Factory 的研究、审查、风控与复盘能力。

---

## 0. 设计定位

本产品的 Web 端不应被设计成传统行情看板，也不应优先设计成机构级研究后台。当前阶段的首要用户是个人投资研究者，他们最直接的高频需求是：

```text
我现在该关注哪些股票？
这些股票来自哪个策略？
什么价格触发买入提醒？
什么价格代表信号失效？
什么价格需要止盈或复盘？
系统什么时候提醒我？
这个信号的可信度和风险如何？
```

因此，Web 端的表层产品定位是：

> 个人策略信号与交易计划助手。

底层仍然是 Personal Alpha Factory OS：

```text
Data → Logic → Signal → Decision → Trade Plan → Alert → Action → Feedback → Control → Evolution
```

但前端必须避免让用户感觉自己在使用复杂机构后台。前端语言要个人化、行动化、提醒化：

```text
今日信号
候选股票
交易计划
到价提醒
风险提醒
策略回测
Alpha 池
AI 助手
```

产品要让用户感觉：

> 我不是在盲目听 AI 荐股，而是在用经过回测和审查的策略规则，管理自己的股票候选、价格计划和提醒。

## 1. 设计原则

### 1.1 表层行动化，底层研究化

用户表层看到的是：

```text
股票候选
触发价
计划区间
止损价
止盈规则
提醒状态
风险边界
```

系统底层保留：

```text
策略来源
回测结果
审查评分
数据版本
信号逻辑
Decision Object
Action Log
Feedback Event
```

前端必须能让用户快速行动，但每一个行动都能追溯到研究依据。

### 1.2 不做“AI 荐股神器”

禁止产品文案：

```text
强烈推荐买入
必涨
稳赚
AI 推荐股票
目标收益保证
```

推荐产品文案：

```text
策略信号
候选股票
触发条件
计划区间
风险边界
信号失效
到价提醒
策略规则生成
```

核心表达：

> 以下计划由你选择的策略规则生成，仅用于研究、提醒和模拟复盘，不构成投资建议。

### 1.3 默认提醒，不默认交易

P0 不接实盘交易，不自动下单。系统允许：

```text
生成候选股票
生成交易计划
设置价格提醒
加入观察列表
记录模拟交易
生成复盘报告
```

系统不允许：

```text
自动买入
自动卖出
自动修改资金仓位
自动推荐用户实盘执行
```

### 1.4 先回答“今天该看什么”

首页不是展示研究成果，而是每日行动面板。第一屏必须回答：

```text
今天有哪些策略触发信号？
哪些股票进入候选？
哪些价格提醒快到了？
哪些计划需要确认？
哪些持仓/观察对象出现风险？
```

### 1.5 所有信号必须有可信度与风险边界

任何股票候选都必须展示：

```text
来源策略
信号类型
触发条件
当前状态
可信度评分
风险等级
买入触发价
信号失效价
提醒设置状态
```

没有信号来源的股票，不应进入“候选股票”主列表。

## 2. 全局布局

Web 端采用三栏布局：

```text
┌──────────────┬──────────────────────────────┬──────────────────┐
│ 左侧导航      │ 中央主工作区                   │ 右侧 AI 助手       │
│ 180–220px    │ 自适应                         │ 320–380px         │
└──────────────┴──────────────────────────────┴──────────────────┘
```

### 2.1 左侧导航

P0 导航建议：

```text
我的工作台
策略信号
股票候选
交易计划
价格提醒
策略回测
Alpha池
研究报告
设置
```

导航含义：

| 页面 | 用户问题 |
|---|---|
| 我的工作台 | 今天我该看什么？ |
| 策略信号 | 今天哪些策略触发了？ |
| 股票候选 | 哪些股票值得加入观察？ |
| 交易计划 | 具体什么价格提醒、止损、止盈？ |
| 价格提醒 | 哪些提醒等待触发或已经触发？ |
| 策略回测 | 这个策略是否可信？ |
| Alpha池 | 我的策略资产和信号源状态如何？ |
| 研究报告 | 有哪些可复盘的报告和记录？ |
| 设置 | 数据源、AI点数、提醒、免责声明等配置 |

P1 可新增：

```text
研究想法
因子研究
模拟组合
团队空间
```

P0 不建议把“研究想法”和“因子研究”放在一级导航前部。个人用户第一入口应是信号、候选、交易计划和提醒。

### 2.2 中央主工作区

中央区域遵循对象优先：

```text
Signal
CandidateStock
TradePlan
PriceAlert
Strategy
Alpha
Report
```

每个页面顶部必须有：

```text
页面标题
一句话说明
全局状态 chips：数据状态 / AI点数 / 风险状态 / 提醒状态
主要操作按钮
```

### 2.3 右侧 AI 助手

右侧不是泛聊天框，而是“当前对象副驾驶”。固定结构：

```text
AI研究助手
├── 当前聚焦对象
├── 当前状态
├── 诊断摘要
├── 推荐下一步
├── AI 点数与成本提示
├── 对话区
└── 快捷指令输入框
```

右栏要随页面上下文切换：

| 页面 | AI 助手重点 |
|---|---|
| 我的工作台 | 今日行动摘要 |
| 策略信号 | 解释信号来源与可信度 |
| 股票候选 | 解释候选原因与风险边界 |
| 交易计划 | 生成/检查买入、止损、止盈和提醒规则 |
| 价格提醒 | 解释提醒触发与后续处理 |
| 策略回测 | 审查回测可信度 |
| Alpha池 | 解释策略状态变化 |
| 研究报告 | 生成与复盘报告 |

## 3. 视觉设计系统

### 3.1 总体风格

关键词：

```text
冷静
清晰
可信
个人化
行动导向
低噪音
高信息密度
```

避免：

```text
赌博感
荐股群感
红绿大屏感
机构后台压迫感
过度炫技图表
```

### 3.2 色彩建议

主色：

```text
Primary Blue: #2563EB
Primary Dark: #1E3A8A
Background: #F8FAFC
Card: #FFFFFF
Border: #E5E7EB
Text Primary: #0F172A
Text Secondary: #64748B
```

状态色：

```text
Success / 已满足: #10B981
Warning / 等待触发: #F59E0B
Danger / 信号失效: #EF4444
Info / 观察中: #3B82F6
Purple / 模拟中: #8B5CF6
Neutral / 已结束: #94A3B8
```

A 股涨跌颜色可按用户市场偏好配置。默认建议减少强烈红绿刺激，重点用状态标签表达风险。

### 3.3 字体与密度

桌面端建议：

```text
页面标题：24–28px / 600
区块标题：16–18px / 600
卡片标题：14–16px / 600
正文：13–14px
表格：12–13px
状态标签：11–12px
```

产品要支持个人投资者长时间使用，避免过度拥挤，但允许高信息密度。

## 16. 合规与风险文案

所有与价格、买卖、计划相关页面必须使用低承诺表达。

```text
必买价格
```

```text
AI 生成内容和策略信号仅供研究、模拟和提醒使用，不构成投资建议、收益承诺或任何形式的交易指令。投资决策需由用户独立判断并自行承担风险。
```

## 19. 关键验收标准

P0 前端完成后，用户必须能够完成以下任务：

```text
1. 打开首页，看见今日需要关注的策略信号和价格提醒。
2. 进入策略信号页，查看哪些策略产生了哪些股票信号。
3. 进入股票候选页，理解某只股票为什么进入候选。
4. 生成一个交易计划，包含触发价、止损价、止盈规则和仓位上限。
5. 为交易计划设置价格提醒。
6. 在提醒触发后查看触发原因和后续处理建议。
7. 查看策略回测与审查结果，理解信号可信度来源。
8. 通过右侧 AI 助手解释信号、生成计划或设置提醒。
9. 在关键页面看到明确免责声明。
```

如果用户只能看到图表，而无法完成“信号 → 候选 → 计划 → 提醒”，则 P0 不合格。

## 20. 最终设计判断

新版 Web 端必须从“研究工作台”升级为“行动工作台”。

底层产品仍然是：

```text
Personal Alpha Factory OS
```

但用户表层体验必须是：

```text
用我的策略筛出股票
把股票变成交易计划
把计划变成价格提醒
把提醒变成复盘记录
```

一句话设计目标：

> 让个人用户每天打开产品后，立即知道哪些股票来自哪些策略、什么价格需要提醒、什么风险边界不能突破，以及下一步该做什么。
